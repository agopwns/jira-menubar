# v2.2.1 팀 온보딩 릴리스 기록

- 완료일: 2026-07-13
- 기능 커밋: [`535c7fb`](https://github.com/agopwns/jira-menubar/commit/535c7fbaccf9cd1ad1fed64065afeb226393ecd2)
- 대상 브랜치: `main`

## 목적

Dev 팀원이 저장소를 받은 뒤 개인 Jira API 토큰과 팀 프로젝트 정보만 입력해 SwiftBar 플러그인을 사용할 수 있도록 설치와 온보딩 과정을 정리했다.

## 주요 변경

- 대화형 설치 프로그램
  - Jira 사이트 URL, 이메일, 숨김 API 토큰, 프로젝트 키, 선택적 보드 ID 입력
  - Jira 계정과 프로젝트 접근 권한 확인
  - `myAccountId` 자동 조회 및 설정 저장
  - 기존의 완전한 설정 보존 및 `--reconfigure` 지원
  - 로컬 체크아웃에서 실행할 경우 현재 체크아웃의 플러그인 설치
- 온보딩 문서
  - [한국어 가이드](../ONBOARDING.md)
  - [English guide](../ONBOARDING.en.md)
- 보안 및 안정성
  - 입력 토큰을 터미널에 표시하지 않고 프로세스 인자에도 넣지 않음
  - 인증 정보를 전송할 주소를 HTTPS `*.atlassian.net`으로 제한
  - 설정·캐시 파일 `600`, 전용 디렉터리 `700`
  - 설치 전 Node 문법, 설정 JSON, 계정 및 프로젝트 접근 검증
  - 임시 파일에 준비한 뒤 최종 파일 교체

## 설치

SwiftBar를 처음 실행해 플러그인 폴더를 지정한 후 다음 명령을 실행한다.

```bash
/bin/bash -o pipefail -c 'curl -fsSL https://raw.githubusercontent.com/agopwns/jira-menubar/main/install.sh | /bin/bash'
```

현재 구현은 Jira 사이트 직접 인증 방식이므로 **Create API token**으로 만든 unscoped API token을 사용한다. Scoped token은 아직 지원하지 않는다.

## 검증 결과

- macOS 기본 Bash 3.2에서 `bash -n install.sh` 통과
- `node --check jira-tickets.5m.js` 통과
- 신규 대화형 설치와 `accountId`·프로젝트·보드 저장 확인
- 토큰 입력 비표시 확인
- 재설치 시 기존 설정 보존 확인
- 로컬 설치와 GitHub `main` 원격 설치 확인
- 플러그인 `755`, 설정·캐시 파일 `600`, 전용 디렉터리 `700` 확인
- README와 한국어·영문 가이드의 설치 및 재설정 명령 일치 확인
- 실제 토큰·이메일·사내 Jira 주소가 저장소에 포함되지 않았음을 확인

## 알려진 제한

- Scoped API token과 OAuth 2.0은 지원하지 않는다.
- API 토큰은 macOS Keychain이 아니라 권한 `600`인 설정 파일에 평문으로 저장된다.
- 보드 ID는 숫자 형식만 확인하며 실제 보드 접근 실패 시 스프린트 통계만 표시되지 않는다.
